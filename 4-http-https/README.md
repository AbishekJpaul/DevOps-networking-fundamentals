# HTTP and HTTPS

## What is HTTP?

HTTP (HyperText Transfer Protocol) is an application layer protocol used for communication between web browsers and web servers.

### Features

- Stateless protocol
- Uses TCP
- Default Port: 80
- Data is transmitted in plain text

### HTTP Workflow

1. User enters URL.
2. Browser sends HTTP request.
3. Server processes the request.
4. Server sends HTTP response.
5. Browser displays the webpage.

### HTTP Methods

- GET
- POST
- PUT
- DELETE
- PATCH

### Drawbacks

- Not secure
- Data can be intercepted
- Vulnerable to attacks

---

## What is HTTPS?

HTTPS (HyperText Transfer Protocol Secure) is the secure version of HTTP.

### Features

- Uses SSL/TLS encryption
- Secure communication
- Default Port: 443
- Protects sensitive information

### SSL/TLS Handshake

1. Client Hello
2. Server Hello
3. Certificate Exchange
4. Key Exchange
5. Secure Connection Established

### Benefits

- Data encryption
- Authentication
- Integrity
- Improved SEO rankings

### Use Cases

- Banking Applications
- E-Commerce Websites
- Social Media Platforms
- Cloud Services

---

## HTTP vs HTTPS

| Feature | HTTP | HTTPS |
|--------|------|-------|
| Security | No | Yes |
| Port | 80 | 443 |
| Encryption | No | Yes |
| SSL/TLS | No | Yes |
| SEO | Lower | Better |

## Conclusion

HTTP is suitable for non-sensitive communication, whereas HTTPS ensures secure and encrypted communication over the internet.
